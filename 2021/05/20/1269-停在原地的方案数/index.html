1269.停在原地的方案数

Posted on 2021-05-20

https://leetcode-cn.com/problems/number-of-ways-to-stay-in-the-same-place-after-some-steps/

有一个长度为 arrLen 的数组，开始有一个指针在索引 0 处。

每一步操作中，你可以将指针向左或向右移动 1 步，或者停在原地（指针不能被移动到数组范围外）。

给你两个整数 steps 和 arrLen ，请你计算并返回：在恰好执行 steps 次操作以后，指针仍然指向索引 0 处的方案数。

由于答案可能会很大，请返回方案数 模 10^9 + 7 后的结果。

示例 1：

输入：steps = 3, arrLen = 2
输出：4
解释：3 步后，总共有 4 种不同的方法可以停在索引 0 处。
向右，向左，不动
不动，向右，向左
向右，不动，向左
不动，不动，不动

优化前

关键点

明显的，像这种要在上一步的基础上前进后退的，基本上就可以用动态规则的思想来解决。

而动态规则最关键的，就是要想清楚 dp 里存什么值，有什么含义。

在这里，题目中说到了求方案数，最直观的能想到用 dp[i] 来表示 steps = i 时的方案数。同时又提到了 dp[i] 和 dp[i-1] 的关系（每一步操作中，你可以将指针向左或向右移动 1 步，或者停在原地），加上这个条件，就有点麻烦了，怎么来表示呢？

感觉要画个图可能比较好理解，比如说：

假如我们要求的是经过 steps = 4 时（*其实这时 steps 是多少并不重要*），
停在下标为 3 的方案数

              dp[4]
由条件 [每一步操作中，你可以将指针向左或向右移动 1 步，或者停在原地]

所以 dp[3] 此时是有 3 种可能的情况
[1]         dp[3] 向右移动一位即可停在 index = 3 处

可见一维数组无法表达这种关系，我们还需要一个维度 [下标]，用来表示进行了 steps 步后，
停在了 index 处的方案数，这样就可以表示 dp[i] 和 dp[i-1] 之间的关系了。

如下：

dp[steps][index] 表示进行了 steps 步后，停在了 index 处
则
在情况 [1] 中，dp[steps][index] = dp[steps-1][index-1]
在情况 [2] 中，dp[steps][index] = dp[steps-1][index]
在情况 [3] 中，dp[steps][index] = dp[steps-1][index+1]
而所有可能方案数即为这3种情况之和
即：
dp[steps][index] = dp[steps-1][index-1] + dp[steps-1][index] + dp[steps-1][index+1]
注意：index-1 和 index+1 不能溢出，还有就是全部方案数的和不能溢出

以上是解决此问题的核心思路，接下来还需要关注一些边界条件。然后我们最终需要的值，其实是 dp[steps][0]，以此思路实现的代码即为上面优化前的代码。

  • 当没有进行任何操作时，指针一定位于下标 0，因此动态规划的边界条件是 dp[0][0] = 1，通俗点说，就是不进行任何(steps=0)操作，同时要索引指在 index = 0 的位置，就只有一种可能
  • 当 1 ≤ j ≤ min(arrLen−1,steps) 时有 dp[0][j] = 0，通俗点说，就是想不进行任何操作，同时又想索引指在 非0 的位置，是不可能的
  • 由于 dp 的第二个维度表示的是下标，又可以得出一个边界，那就是小标应该 arrLen - 1 和 steps 间小的那一个

  • 关于对方案数求模的问题，为了避免溢出，其实是应该对每个状态的方案数都取模，然后再相加的。

    这段代码如何理解？

dp[i][j] =  dp[i-1][j];
if (j - 1 >= 0) { // 保证下标不溢出
    dp[i][j] = (dp[i-1][j-1] + dp[i][j]) % mode;
}
if (j + 1 <= maxIndex) { // 保证下标不溢出
    dp[i][j] = (dp[i-1][j+1] + dp[i][j]) % mode;
}

验证下它的正确性

dp[steps][index] = dp[steps-1][index-1] + dp[steps-1][index] + dp[steps-1][index+1]
简写如下：
dp[s][i] = dp[s-1][i-1] + dp[s-1][i] + dp[s-1][i+1]
则 
方案数 count = (dp[s-1][i-1] + dp[s-1][i] + dp[s-1][i+1]) % m

代码刚开始时
dp[i][j] =  dp[i-1][j]
然后
dp[i][j] = (dp[i-1][j-1] + dp[i][j]) % mode
         = (dp[i-1][j-1] + dp[i-1][j]) % mode
然后
dp[i][j] = (dp[i][j] + dp[i-1][j+1]) % mode
         = ((dp[i-1][j-1] + dp[i-1][j]) % mode + dp[i-1][j+1]) % mode
mode 记作 m
记 dp[i-1][j-1] = a
   dp[i-1][j] = b
   dp[i-1][j+1] = c，则有
方案数 count = ((a + b) % m + c) % m  

实际不考虑溢出的话，我们求的是 (a + b + c) % m，看结果这里的值是相等的，也就是说，
需要证明 (a + b) % m + c 和 a + b + c 是同余式，
简化一下，记 a+b = a, c = b，即需要证明
a % m + b ≡ a + b (mod m) 即 a % m + b 和 a + b 是以 m 为模的同余式

how?

这里涉及一个同余式的概念

设m是大于1的正整数，a、b是整数，如果m|(a-b)，则称a与b关于模m同余，记作a≡b(mod m)，读作a与b对模m同余。
记作：a≡b (mod m)，
读作：a同余于b模m，或读作a与b对模m同余，例如26≡2(mod 12)。
同时，同余式还有一个非常重要的性质，即：
（1）若a≡0(mod m)，则m|a；
（2）a≡b(mod m)等价于a与b分别用m去除，余数相同

根据 m|(a-b)和性质(1) ，能证明 (a + b) % m + c - (a + b + c) ≡ 0(mod m) 即可

假设 (a + b) % m + c - (a + b + c) ≡ 0(mod m)
(a + b) % m + c - (a + b + c)
    = (a + b) % m - (a + b)
记 a + b = a
也就是要证明 a % m - a ≡ 0(mod m)
即证明 a % m - a = n * m (n >= 0)，也就是证明 a % m  - a 能被 m 整除
这是肯定的。
因为对任意一个数 a, 关于 m 的余数，都有以下关系
a % m = a - n*m
所以 
a % m - a = -n*m
-n*m ≡ 0(mod m)

其实这里的 dp[i][j] 就是原 dp[i][j] 取模后的值

从 dp[steps][index] = dp[steps-1][index-1] + dp[steps-1][index] + dp[steps-1][index+1] 可以看到，当前的 dp[stesp] 只和它的上一个(dp[steps-1])有关，因此可以用一个一维数组。和上面的相比，上面用二维数组是保存了step 从 0 到 1 的全部方案数，现在只需要保存上一步的方案数即可，相当于是取的二维数组的第二个维度。
