494.目标和

Posted on 2021-06-07 Edited on 2021-06-13

https://leetcode-cn.com/problems/target-sum/

给你一个整数数组 nums 和一个整数 target 。

向数组中的每个整数前添加 ‘+’ 或 ‘-‘ ，然后串联起所有整数，可以构造一个 表达式 ：

例如，nums = [2, 1] ，可以在 2 之前添加 ‘+’ ，在 1 之前添加 ‘-‘ ，然后串联起来得到表达式 “+2-1” 。
返回可以通过上述方法构造的、运算结果等于 target 的不同 表达式 的数目。

示例 1：

输入：nums = [1,1,1,1,1], target = 3
输出：5
解释：一共有 5 种方法让最终目标和为 3 。
-1 + 1 + 1 + 1 + 1 = 3
+1 - 1 + 1 + 1 + 1 = 3
+1 + 1 - 1 + 1 + 1 = 3
+1 + 1 + 1 - 1 + 1 = 3
+1 + 1 + 1 + 1 - 1 = 3

示例 2：

输入：nums = [1], target = 1
输出：1

提示：

  • 1 <= nums.length <= 20
  • 0 <= nums[i] <= 1000
  • 0 <= sum(nums[i]) <= 1000
  • -1000 <= target <= 100

  • 时间复杂度：O(2^n)

解法二：动态规划

基本上当前结果和上一步有关的这种题目，都可以往动态规划上靠。

动态规划主要思考的关键点有两个：

  1. dp 数组：就是如何来定义这个数组？用几组数组？各维度分别代表什么？
  2. 状态转移方程：
    a). 初始值
    b). 如何从上一个元素得到当前的值

dp 数组

对于这个题目，定义二维数组 dp，其中 dp[i][j] 表示在数组 nums 的前 ii 个数中选取元素，使得这些元素之和等于 j 的方案数。假设数组 nums 的长度为 n，则最终答案为 dp[n][neg].

状态转移方程

初始条件：

当没有任何元素可以选取时，元素和只能是 0，对应的方案数是 1，因此动态规划的边界条件是：dp[0][0] = 1

记数组和为 sum，满足条件的数组中，带 “-” 的元素和为 neg，由于数组中的元素都是大于 0 的，所以 neg 也是大于0。根据题目要求有：

则带 “+” 的元素和为 sum - neg。
target = (sum - neg) - neg   好好理解一下
       = sum - 2neg

neg = (sum - target) / 2

由于 neg > 0，并且 neg 是整数，
因此： sum - target > 0，并且是 2 的整数倍

对于 dp[i][j]，也就是前 i 个元素之和等于 j 的方案数。

  1. dp[i-1][j] 也就是前 i-1 个元素之和等于 j，这时由于 nums[i] > 0 的，所以 dp[i][j]，也就是前 i 个元素之和，明显的dp[i-1][j] 不管是 +nums[i] 还是 -nums[i] 都不可能再等于 j 了，所以在这个维度， dp[i][j] = dp[i-1][j] 的，也就是 dp[i][j] 至少是等于 dp[i-1][j] 的
  2. 然后对于第二个维度，也就是 dp[i-1][j-x] 这个维度上，也就是说对于现在的 nums[i]，我们要在 dp[i-1][xxx] 这个维度中，找到一个值，tagarg = j，也就是要找的是 j - nums[i] 这个下标的方案数，就是 dp[i][j] 的方案数了。

  3. 为什么要 j >= nums[i-1]？

    因为你想，我们需要求的是 j 对吧，那如果 nums[i-1] 都大于 j 了，j-nums[i-1] 就小于 0 了，那前面就不可能存在一个下标和 nums[i-1] 相加得到 j 了

所以上面的代码表示 dp[i][j] 的值至少是 dp[i - 1][j]，同时当 j >= nums[i-1] 时，还需要再加上 dp[i-1][j-nums[i-1]] 的方案数。

解法三：动态规划

一般动态规划都可以尝试减少数组维度，比如把一维 dp 变成一个变量值，把二维 dp 变成一个一维 dp 数组。

可以看到我们始终只用了 dp[i-1]，也就是 dp[i] 的上一个行的数组。因为只需要用一个一维数组，表示 dp[i] 的上一行即可。可优化为下面：

为什么这里要倒序？

上面有提到，对于现在的 dp数组来说，当 i 增加，相应于之前是遍历求 dp[i][xxx] ，dp数组应该要表示的是 dp[i-1][xxx] 的值，如果在这里我们顺序的来给 dp[j] 赋值，那 dp[j+1] 的值就被污染了，因为 dp[j+1] 不再是由 dp[i-1][j] 得到的，而是由 dp[i][j] 得到的了。而从后往前遍历就不会污染 dp数组的值了。

为什么这里只需要遍历到 nums[i-1]？

因为这里是个一维的复用的数组了， dp[j] 此时不再是初始值，而是本身就相当于是之前的 dp[i-1][j] 了，仔细体会。
